Java Inheritance


In Java, inheritance is a mechanism by which a class (known as a subclass or child class) can acquire properties and behaviors (methods and fields) from another class (known as a superclass or parent class). This enables code reuse, promotes modularity, and facilitates the creation of hierarchical relationships among classes.

Here is detailed information of Inheritance :

Defination :

Inheritance in Java is the process by which a class inherits features (fields and methods) from another class, allowing the subclass to reuse and extend the functionality defined in the superclass. The subclass inherits the properties and behaviors of the superclass and can add new features or modify existing ones as needed.

Key Points

  1. Superclass and Subclass Relationship :
    Inheritance establishes a hierarchical relationship between classes, where the subclass extends the superclass. The superclass provides a common blueprint, and subclasses refine or specialize the behavior of the superclass.

  2. Syntax :
    In Java, inheritance is implemented using the extends keyword. The subclass class declaration specifies the superclass it inherits from.
    3. 
class Subclass extends Superclass {
    // subclass members
}

  3. Access to Superclass Members :
    Subclasses inherit the public and protected members of the superclass. Private members are not directly accessible in subclasses. Package-private members are accessible within the same package.

  4. Code Reusability :
    Inheritance promotes code reuse by allowing subclasses to inherit and reuse the behavior and attributes defined in the superclass. This reduces redundancy and leads to more maintainable and scalable code.

  5. Method Overriding :
    Subclasses can override methods inherited from the superclass to provide specialized implementations. This allows subclasses to tailor the behavior of inherited methods to suit their specific requirements.

  6. Inheritance Hierarchy :
    In Java, classes can form a hierarchy of inheritance, where subclasses can further serve as superclasses for other subclasses. This creates a tree-like structure of classes, with each level of the hierarchy inheriting features from its parent classes.

  7. Single Inheritance :
    Java supports single inheritance, meaning a class can inherit from only one superclass. However, it can implement multiple interfaces, enabling multiple inheritance of behavior.

  8. Polymorphism :
    Inheritance facilitates polymorphism, where objects of the subclass can be treated as objects of the superclass, enabling dynamic method dispatch and runtime polymorphic behavior.


Different types of Inheritance :




  1. Single Inheritance :

    In single inheritance, a subclass inherits from only one superclass. This is the simplest form of inheritance. 
    
class Superclass {
    // superclass members
}
    
class Subclass extends Superclass {
    // subclass members
}

    characteristics :
    • Each subclass has exactly one direct superclass.

    • Java supports single inheritance for classes, meaning a class can extend only one other class.

    Example Use Case :
    • Modeling relationships where a subclass represents a more specific type of the superclass. For instance, a ' Car ' class (subclass) might inherit from a ' Vehicle ' class (superclass).

  2. Multilevel Inheritance :

    In multilevel inheritance, a subclass inherits from another subclass, creating a chain of inheritance. 
     
class Grandparent {
    // grandparent members
}
    
class Parent extends Grandparent {
    // parent members
}
    
class Child extends Parent {
    // child members
}

    characteristics :
    • Classes are organized in a hierarchical order, with each subclass inheriting from its immediate parent class.

    • The subclass at the end of the chain inherits properties and behaviors from all its ancestor classes.

    Example Use Case :
    • Representing relationships where each subsequent subclass adds additional features or behavior to the preceding ones

  3. Hierarchical Inheritance :

    In hierarchical inheritance, multiple subclasses inherit from the same superclass. 
    
class Superclass {
    // superclass members
}
    
class Subclass1 extends Superclass {
    // subclass1 members
}
    
class Subclass2 extends Superclass {
    // subclass2 members
}

    characteristics :
    • Multiple subclasses inherit from a single superclass.

    • Each subclass shares common features defined in the superclass but may have its own unique characteristics.

    Example Use Case :
    • Modeling scenarios where different subclasses share common properties or behavior but also have their distinct features. For example, a ' Shape ' superclass with subclasses like ' Rectangle ', ' Circle ', and ' Triangle '.

  4. Multiple Inheritance (Through Interfaces) :

    Java doesn't support multiple inheritance for classes, where a class inherits from more than one superclass. However, it allows multiple inheritance of behavior through interfaces. 
    
interface Interface1 {
    // interface1 members
}
    
interface Interface2 {
    // interface2 members
}
    
class MyClass implements Interface1, Interface2 {
    // class members
}

    characteristics :
    • A class can implement multiple interfaces, effectively inheriting behavior from all of them.

    • This allows a class to exhibit the characteristics of multiple types without requiring multiple inheritance of classes.

    Example Use Case :
    • Enabling a class to provide various functionalities defined in different interfaces, promoting code modularity and flexibility.
