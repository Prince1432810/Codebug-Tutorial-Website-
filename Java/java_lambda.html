Lambda Expressions In Java


Lambda expressions in Java are a powerful feature introduced in Java 8. They allow you to write more concise and expressive code, especially when working with functional interfaces. Lambda expressions provide a way to represent anonymous functions—functions without a name—that can be passed as arguments or stored in variables.


Syntax :

The syntax of a lambda expression consists of three parts:

  1. Parameter List :
    A lambda expression can have zero or more parameters. If there are no parameters, empty parentheses are used. For multiple parameters, parentheses are required. For example:

    • No parameters : ' () -> System.out.println("Hello, World!"); '
    • Single parameter : ' (int x) -> x * x; '
    • Multiple parameters : ' (int x, int y) -> x + y; '

  2. Arrow Token :
    The arrow token -> separates the parameter list from the body of the lambda expression. It is read as "goes to." For example: ' (x, y) -> x + y; '

  3. Body :
    The body of the lambda expression can be a single expression or a block of code. If the body is a single expression, the return type is inferred from the context. If the body is a block of code, curly braces {} are used, and a return statement may be required. For example:

    • Single expression : ' x -> x * x; '
    • Block of code : ' (x, y) -> { int sum = x + y; return sum;} '
    4. (parameters) -> expression
    (parameters) -> { statements; }

    Example :
    Runnable runnable = () -> System.out.println("Hello, World!");

Functional Interface In Java


Lambda expressions are often used in conjunction with functional interfaces, which are interfaces that have exactly one abstract method. Java provides many built-in functional interfaces in the ' java.util.function ' package, such as ' Predicate ', ' Function ', ' Consumer ', and ' Supplier '.

For example, the ' Predicate ' interface represents a boolean-valued function of one argument. You can use a lambda expression to implement this interface as follows:

Predicate isPositive = x -> x > 0;
    System.out.println(isPositive.test(5)); // true
    System.out.println(isPositive.test(-5)); // false

Characteristics of Functional Interfaces


  1. Single Abstract Method :
    Functional interfaces must have only one abstract method. However, they can have multiple default or static methods.

  2. Annotation :
    While not required, it's a good practice to annotate functional interfaces with ' @FunctionalInterface ' to ensure they are treated as functional interfaces by the compiler. This annotation is not mandatory but helps prevent accidental addition of more than one abstract method in the future.

  3. Inheritance :
    Functional interfaces can extend other interfaces, and if they meet the criteria of having only one abstract method after inheritance, they are still considered functional interfaces.

Type Inference


In many cases, the compiler can infer the type of parameters in a lambda expression based on the context. For example, in the ' Predicate ' example above, the type of ' x ' is inferred to be ' Integer ' because ' Predicate<Integer> ' is specified.


Capturing Variables

Lambda expressions can capture variables from their enclosing scope. These variables must be effectively final, which means they are not reassigned after being initialized. For example:

int y = 2;
    Function adder = x -> x + y;
    System.out.println(adder.apply(3)); // 5

In this example, ' y ' is captured from the enclosing scope and used in the lambda expression.


Conclusion

Lambda expressions in Java provide a concise way to represent anonymous functions, making your code more readable and expressive. They are commonly used in functional programming and with streams to process collections of data.
