Annotations In Java


Annotations in Java are a form of metadata that provide data about a program but are not part of the program itself. They can be added to Java source code to provide information to the compiler, to tools, or to runtime environments. Annotations have various uses, such as providing additional information to the compiler, marking methods as deprecated, or configuring frameworks.


Anatomy of an Annotation

An annotation in Java consists of an ' @ 'symbol followed by the annotation name and, optionally, a list of key-value pairs called elements. Annotations can be applied to classes, methods, fields, parameters, and other program elements.

@AnnotationName(key1=value1, key2=value2)
  public class MyClass {
      // Class body
  }

Built-in Annotations

Java provides several built-in annotations that serve different purposes. Some common ones include:

  • @Override : Indicates that a method overrides a method in its superclass.
  • @Deprecated : Marks a program element (class, method, field, etc.) as deprecated, indicating that it should no longer be used.
  • @SuppressWarnings : Suppresses compiler warnings for the annotated element.
  • @FunctionalInterface : Indicates that an interface is a functional interface.

Custom Annotations

Java also allows you to define your own annotations, known as custom annotations. To define a custom annotation, you use the ' @interface ' keyword followed by the annotation name and optional elements.

import java.lang.annotation.ElementType;
  import java.lang.annotation.Retention;
  import java.lang.annotation.RetentionPolicy;
  import java.lang.annotation.Target;
  
  @Retention(RetentionPolicy.RUNTIME)
  @Target(ElementType.METHOD)
  public @interface MyAnnotation {
      String value() default "default value";
      int count() default 1;
  }

In this example, ' @Retention ' specifies how long the annotation should be retained. ' @Target ' specifies where the annotation can be applied. The ' MyAnnotation ' annotation has two elements: ' value ' and ' count ', both with default values


Using Annotation

Annotations can be used at compile time by the compiler, at build time by build tools (e.g., Maven, Gradle), or at runtime by the JVM or other libraries/frameworks. They can influence the behavior of the program or provide additional information for tools and developers.

For example, the Spring framework uses annotations extensively to configure beans, define request mappings, and manage transactions, among other things.

@Service
  public class MyService {
      @Autowired
      private MyRepository repository;
  
      @Transactional
      public void doSomething() {
          // Method body
      }
  }

In this example, ' @Service ', ' @Autowired ', and ' @Transactional ' are annotations used by the Spring framework to configure the ' MyService ' class.

Annotations are a powerful feature of the Java language that allow you to add metadata to your code, which can be used by the compiler, tools, and frameworks to improve development and runtime behavior
