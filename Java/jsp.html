JavaServer Pages


JavaServer Pages (JSP) is a technology used to create dynamic web content in Java web applications. JSP allows developers to embed Java code in HTML pages, making it easier to create dynamic web pages compared to servlets alone. JSP pages are compiled into servlets by the web container (e.g., Tomcat, Jetty) and executed on the server side to generate dynamic content that is sent to the client (e.g., web browser).


How JSP Work


  1. JSP Compilation :
    When a JSP page is requested for the first time, the web container compiles it into a servlet. This servlet is responsible for generating the dynamic content of the JSP page.

  2. Servlet Execution :
    Each time a client requests the JSP page, the servlet is executed. The servlet processes any embedded Java code, generates the HTML content, and sends it back to the client.

  3. Dynamic Content Generation :
    JSP allows developers to embed Java code directly into the HTML page using special tags (<% %>, <%= %>, <%! %>) and expressions (${}) to generate dynamic content.

JSP Tags


JSP provides several tags that can be used to perform different actions and control the flow of the page. Some of the commonly used JSP tags include:

  • Directives :
    Used to provide instructions to the container about how to process the JSP page. For example, the <%@ page %> directive is used to set page-specific attributes.

  • Declarations :
    Used to declare variables and methods that can be used later in the JSP page. Declarations are enclosed in <%! %> tags.

  • Scriptlets :
    Used to write Java code that is embedded directly into the servlet generated from the JSP page. Scriptlets are enclosed in <% %> tags.

  • Expressions :
    Used to output the result of a Java expression directly into the HTML content. Expressions are enclosed in <%= %> tags.

  • Actions :
    Provide more complex functionality, such as including other files (<jsp:include>), forwarding requests (<jsp:forward>), or iterating over collections (<jsp:useBean>).

Example of JS Page


<%@ page contentType="text/html;charset=UTF-8" language="java" %>
    <html>
    <head>
        <title>My JSP Page</title>
    </head>
    <body>
        <h1>Hello, <%= request.getParameter("name") %>!</h1>
        <% for (int i = 0; i < 5; i++) { %>
            <p>This is paragraph <%= i+1 %></p>
        <% } %>
    </body>
    </html>

In this example, the JSP page uses the <%= %> expression to output the value of the name parameter from the request. It also uses a scriptlet (<% %>) to loop and generate five paragraphs.


JSP Lifecycle


  1. Translation :
    The JSP page is translated into a servlet by the JSP container.

  2. Compilation :
    The servlet is compiled into Java bytecode.

  3. Initialization :
    The servlet is instantiated and its init() method is called.

  4. Request Handling :
    Each client request results in the servlet's service() method being called to generate the response.

  5. Destruction :
    When the servlet container decides to remove the servlet, its destroy() method is called to clean up any resources.

Conclusion

JavaServer Pages (JSP) is a technology that simplifies the creation of dynamic web content in Java web applications. By allowing developers to embed Java code directly into HTML pages, JSP makes it easier to create dynamic and interactive web pages. JSP pages are compiled into servlets, which are executed on the server side to generate the final HTML content sent to the client.
