HashSet in Java


In Java, ' HashSet ' is a part of the Java Collections Framework and is an implementation of the ' Set ' interface. It represents a collection that does not allow duplicate elements. The primary features of ' HashSet ' include:


  1. Unordered Collection :
    ' HashSet ' does not guarantee the order of its elements. It does not maintain the order in which the elements are inserted.

  2. Unique Elements :
    ' HashSet ' does not allow duplicate elements. If you attempt to add an element that is already present, the ' add() ' method will return ' false ', indicating that the element was not added.

  3. Backed by a Hash Table :
    Internally, ' HashSet ' uses a hash table to store its elements. This allows for constant-time performance for basic operations like adding, removing, and checking for the presence of elements.

  4. Null Elements :
    ' HashSet ' permits a single null element. It means you can add null to a ' HashSet ' instance once.

  5. Not Synchronized :
    ' HashSet ' is not synchronized. If multiple threads access a ' HashSet ' concurrently, and at least one of the threads modifies the HashSet structurally, it must be synchronized externally.

    6. HashSet is not synchronized. If multiple threads access a HashSet concurrently, and at least one of the threads modifies the HashSet structurally, it must be synchronized externally. 
    import java.util.HashSet;

public class HashSetExample {
    public static void main(String[] args) {
        // Creating a HashSet
        HashSet<String> hashSet = new HashSet<>();
  
        // Adding elements
        hashSet.add("Apple");
        hashSet.add("Banana");
        hashSet.add("Orange");
        hashSet.add("Apple"); // This will not be added because it's a duplicate
  
         // Displaying the HashSet
        System.out.println("HashSet: " + hashSet);
  
        // Removing an element
        hashSet.remove("Banana");
  
        // Checking if an element exists
        System.out.println("Contains Orange? " + hashSet.contains("Orange"));
  
        // Size of the HashSet
         System.out.println("Size of HashSet: " + hashSet.size());
    }
}

    Output : 
    HashSet: [Orange, Banana, Apple]
Contains Orange? true
Size of HashSet: 2

Remember, ' HashSet ' offers constant-time performance for basic operations like add, remove, contains, and size, assuming the hash function disperses the elements properly among the buckets. However, the performance of ' HashSet ' might degrade if many elements hash to the same bucket, resulting in a longer chain of elements. In such cases, ' LinkedHashSet ' might be preferred as it maintains insertion order while still offering constant-time performance.


TreeSet in Java

' TreeSet ' is another implementation of the Set interface in Java, provided by the Java Collections Framework. It maintains a sorted set of elements using a red-black tree. Here are the key points about ' TreeSet ':

  1. Sorted Set :
    Unlike ' HashSet ', which does not guarantee any specific order of elements, ' TreeSet ' maintains the elements in sorted order. This ordering can be the natural ordering of the elements (if they implement the Comparable interface) or a custom ordering provided by a Comparator at the time of ' TreeSet ' creation.

  2. Red-Black Tree :
    Internally, ' TreeSet ' uses a self-balancing binary search tree called a red-black tree to store its elements. This ensures that the operations like add, remove, and search have a time complexity of O(log n), where n is the number of elements in the ' TreeSet '.

  3. No Duplicates :
    Like all implementations of the ' Set ' interface, ' TreeSet ' does not allow duplicate elements. If you attempt to add a duplicate element, the ' add() ' method will simply return ' false ', indicating that the element was not added.

  4. Null Elements :
    ' TreeSet ' does not allow null elements. If you try to add null to a ' TreeSet ', it will throw a ' NullPointerException '.

  5. Not Synchronized :
    Similar to ' HashSet ', ' TreeSet ' is not synchronized. If multiple threads access a ' TreeSet ' concurrently, and at least one of the threads modifies the ' TreeSet ' structurally, it must be synchronized externally

    6. Here's a simple example of how to use ' TreeSet ': 
    import java.util.TreeSet;

public class TreeSetExample {
    public static void main(String[] args) {
        // Creating a TreeSet of strings
        TreeSet<String> treeSet = new TreeSet<>();

        // Adding elements
        treeSet.add("Apple");
        treeSet.add("Banana");
        treeSet.add("Orange");
        treeSet.add("Apple"); // This will not be added because it's a duplicate

        // Displaying the TreeSet
        System.out.println("TreeSet: " + treeSet);

        // Removing an element
        treeSet.remove("Banana");

        // Checking if an element exists
        System.out.println("Contains Orange? " + treeSet.contains("Orange"));

        // Size of the TreeSet
        System.out.println("Size of TreeSet: " + treeSet.size());
    }
}

    output : 
    TreeSet: [Apple, Banana, Orange]
Contains Orange? true
Size of TreeSet: 2

    In summary, ' TreeSet ' is a sorted set implementation in Java that uses a red-black tree for storage and provides efficient operations for maintaining a sorted collection of unique elements.
