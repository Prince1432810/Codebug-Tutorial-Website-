Lists,Sets and Maps in java


In Java, lists, sets, and maps are three fundamental interfaces provided by the Java Collections Framework to store and manipulate collections of objects. Each of these interfaces has multiple implementations tailored to different use cases. Here's an overview of each:

  1. Sets :

    • ' Sets ' in Java represent unordered collections of unique elements. Each element in a set must be unique, and duplicate elements are not allowed.

    • The Set interface extends the Collection interface and defines methods for adding, removing, and querying elements.

    • Key implementations of the Set interface include:
      • HashSet :
        Implements the ' Set ' interface using a hash table for storage, offering constant-time performance for basic operations.
      • TreeSet :
        Implements the ' NavigableSet' interface using a red-black tree for storage, offering sorted order of elements.
      • LinkedHashSet :
        Implements the ' Set ' interface using a hash table with a linked list for storage, offering predictable iteration order.
    • Example : 
      • Set<String> hashSet = new HashSet<>();
hashSet.add("Apple");
hashSet.add("Banana");
hashSet.add("Apple"); // Duplicate element
System.out.println(hashSet); // Output: [Apple, Banana]

  2. Lists :

    • ' Lists ' in Java represent ordered collections of elements. Each element in a list has an index associated with its position in the list.

    • The ' List ' interface extends the ' Collection ' interface and defines methods for accessing, inserting, removing, and manipulating elements based on their index.

    • Key implementations of the List interface include:
      • ' ArrayList: '
        Resizable-array implementation of the ' List ' interface, offering fast random access and dynamic resizing.
      • ' LinkedList : '
        Doubly linked-list implementation of the ' List ' interface, offering fast element insertion and removal, but slower random access.
      • ' Vector : '
        Synchronized version of ' ArrayList ', although its usage is discouraged in favor of ' ArrayList '.
    • Example : 
      • List<String> arrayList = new ArrayList<>();
arrayList.add("One");
arrayList.add("Two");
arrayList.add("Three");
System.out.println(arrayList); // Output: [One, Two, Three]

  3. Maps :

    • Maps in Java represent collections of key-value pairs, where each key is associated with a unique value.

    • The ' Map ' interface defines methods for adding, removing, and querying key-value pairs based on their keys.

    • Key implementations of the ' Map ' interface include:
      • HashMap: Implements the ' Map' interface using a hash table for storage, offering constant-time performance for basic operations.
      • TreeMap:
        Implements the ' NavigableMap ' interface using a red-black tree for storage, offering sorted order of keys.
      • LinkedHashMap:
        Implements the ' Map ' interface using a hash table with a linked list for storage, offering predictable iteration order.
    • Example : 
      • Map<String, Integer> hashMap = new HashMap<>();
hashMap.put("One", 1);
hashMap.put("Two", 2);
hashMap.put("Three", 3);
System.out.println(hashMap); // Output: {One=1, Two=2, Three=3}

These interfaces and their implementations provide powerful tools for organizing and manipulating collections of objects in Java programs, enabling developers to efficiently manage data structures of various types and complexities.
