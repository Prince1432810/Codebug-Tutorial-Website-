Reading Files In Java


Reading files in Java refers to the process of accessing and extracting data from files stored on a computer's disk using Java programming language. This process involves opening a file, reading its contents, and then closing the file. Java provides several classes and methods in its standard library to perform file reading operations, such as ' FileInputStream ', ' FileReader ', ' BufferedReader ', and ' Scanner '. These classes allow you to read various types of files, including text files, binary files, and more, depending on your requirements.


  1. File Input Stream (FileInputStream) :

    • FileInputStream is used to read bytes from a file.

    • It's suitable for reading raw bytes such as images, audio, or binary files.

    • Example:
      • try (FileInputStream fis = new FileInputStream("example.txt")) {
    int data;
    while ((data = fis.read()) != -1) {
        // Process the data
    }
} catch (IOException e) {
    e.printStackTrace();
}

  2. File Reader (FileReader) :

    • FileReader is used to read characters from a file.

    • It's suitable for reading text files.

    • Example:
      • try (FileReader fr = new FileReader("example.txt")) {
    int data;
    while ((data = fr.read()) != -1) {
        // Process the data (which is a character)
    }
} catch (IOException e) {
    e.printStackTrace();
}

  3. Buffered Reader (BufferedReader) :

    • ' BufferedReader ' is used to read text from a character-input stream with efficiency.

    • It's suitable for reading large files efficiently.

    • Example:
      • try (BufferedReader br = new BufferedReader(new FileReader("example.txt"))) {
    String line;
    while ((line = br.readLine()) != null) {
        // Process the line
    }
} catch (IOException e) {
    e.printStackTrace();
}

  4. Scanner :

    • ' Scanner ' is a simple text scanner that can parse primitive types and strings.

    • It's suitable for processing tokens from the input, like parsing a file line by line.

    • Example:
      • try (Scanner scanner = new Scanner(new File("example.txt"))) {
    while (scanner.hasNextLine()) {
        String line = scanner.nextLine();
        // Process the line
    }
} catch (FileNotFoundException e) {
    e.printStackTrace();
}



Writing Files In Java


Writing files in Java involves creating, modifying, or appending data to a file on the disk. Java provides several classes and methods in its standard library to perform file writing operations, such as FileOutputStream, FileWriter, BufferedWriter, and PrintWriter. These classes allow you to write various types of files, including text files, binary files, and more, depending on your requirements


  1. File Output Stream (FileOutputStream) :

    • ' FileOutputStream ' is used to write bytes to a file.

    • It's suitable for writing raw bytes such as images, audio, or binary files.

    • Example:
      • try (FileOutputStream fos = new FileOutputStream("example.txt")) {
    fos.write("Hello, World!".getBytes());
} catch (IOException e) {
    e.printStackTrace();
}

  2. File Writer (FileWriter) :

    • FileWriter is used to write characters to a file.

    • It's suitable for writing text files.

    • Example:
      • try (FileWriter fw = new FileWriter("example.txt")) {
    fw.write("Hello, World!");
} catch (IOException e) {
    e.printStackTrace();
}

  3. Buffered Writer (BufferedWriter) :

    • BufferedWriter is used to write text to a character-output stream with efficiency.

    • It's suitable for writing large files efficiently.

    • Example:
      • try (BufferedWriter bw = new BufferedWriter(new FileWriter("example.txt"))) {
    bw.write("Hello, World!");
} catch (IOException e) {
    e.printStackTrace();
}

  4. Print Writer (PrintWriter) :

    • PrintWriter is used to write formatted text to a file.

    • It provides methods like println for convenient writing of various data types.

    • Example:
      • try (PrintWriter pw = new PrintWriter(new FileWriter("example.txt"))) {
    pw.println("Hello, World!");
} catch (IOException e) {
    e.printStackTrace();
}

    5. When writing files, ensure that you handle exceptions properly and close the file writer or output stream after you finish writing to free up system resources. Use the try-with-resources statement in Java to automatically close the resources when they are no longer needed.
