Servlets In Java


Servlets are Java classes that extend the capabilities of servers to handle requests and responses. They are the backbone of web development in Java, providing a way to handle HTTP requests and generate dynamic web content. Servlets run on the server side and are used to build web applications that respond to client requests.


How Servlets Work


  1. Client Sends Request :
    A client (such as a web browser) sends an HTTP request to the server for a specific URL.

  2. Servlet Container Handles Request :
    The servlet container (such as Tomcat, Jetty, or WildFly) receives the request and determines which servlet should handle it based on the URL mapping defined in the deployment descriptor (web.xml) or using annotations (since Servlet 3.0).

  3. Servlet Processes Request :
    The servlet processes the request, which may involve interacting with databases, processing form data, or generating dynamic content.

  4. Servlet Sends Response :
    The servlet generates an HTTP response, which is sent back to the client.

  5. Client Receives Response :
    The client (web browser) receives the response and displays it to the user.

Servlet Lifecycle


  1. Initialization :
    When a servlet is first loaded, the servlet container initializes it by calling its init() method. This method is called only once during the lifecycle of the servlet.

  2. Request Handling :
    For each client request, the servlet container calls the servlet's service() method. This method determines the type of request (GET, POST, etc.) and calls the appropriate method (doGet(), doPost(), etc.) to handle the request.

  3. Destruction :
    When the servlet container decides to remove the servlet (e.g., when the application is shut down), it calls the servlet's destroy() method to allow the servlet to release any resources it has allocated.

Creating a Servlet


To create a servlet in Java, you need to create a class that extends the HttpServlet class and override the doGet() or doPost() method to handle the request. You also need to configure the servlet in the deployment descriptor (web.xml) or use annotations.

Example 
import java.io.IOException;
    import javax.servlet.http.HttpServlet;
    import javax.servlet.http.HttpServletRequest;
    import javax.servlet.http.HttpServletResponse;
    
    public class MyServlet extends HttpServlet {
    
        protected void doGet(HttpServletRequest request, HttpServletResponse response) throws IOException {
            response.setContentType("text/html");
            response.getWriter().println("<html><body><h1>Hello, World!</h1></body></html>");
        }
    }

Deployment Descriptor (web.xml)


In older versions of Java EE, servlets were configured using the ' web.xml ' deployment descriptor. However, in modern servlet development, annotations are typically used for configuration, making ' web.xml ' optional.

Example web.xml configuration:

<servlet>
    <servlet-name>MyServlet</servlet-name>
    <servlet-class>com.example.MyServlet</servlet-class>
</servlet>
<servlet-mapping>
    <servlet-name>MyServlet</servlet-name>
    <url-pattern>/hello</url-pattern>
</servlet-mapping>

Annotation

Since Servlet 3.0, annotations can be used to configure servlets, making deployment descriptors optional. Annotations are placed directly on the servlet class to specify its URL mapping.

Example using annotations

import javax.servlet.annotation.WebServlet;

    @WebServlet("/hello")
    public class MyServlet extends HttpServlet {
    
        protected void doGet(HttpServletRequest request, HttpServletResponse response) throws IOException {
            response.setContentType("text/html");
            response.getWriter().println("<html><body><h1>Hello, World!</h1>/body></html>");
        }
    }

Conclusion

Servlets are a fundamental part of Java web development, providing a way to handle HTTP requests and generate dynamic web content. They are used to build web applications that run on the server side and interact with clients (such as web browsers) over the internet.
